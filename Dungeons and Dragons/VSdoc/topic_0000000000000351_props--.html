RoomInfo Properties

The following tables list the members exposed by the RoomInfo type.

Properties
 
Name Description
CustomProperties Read-only "cache" of custom properties of a room. Set via Room.SetCustomProperties (not available for RoomInfo class!).
IsLocalClientInside State if the local client is already in the game or still going to join it on gameserver (in lobby always false).
IsOpen Defines if the room can be joined. This does not affect listing in a lobby but joining the room will fail if not open. If not open, the room is excluded from random matchmaking. Due to racing conditions, found matches might become closed before they are joined. Simply re-connect to master and find another. Use property "IsVisible" to not list the room.
IsVisible Defines if the room is listed in its lobby. Rooms can be created invisible, or changed to invisible. To change if a room can be joined, use property: open.
MaxPlayers Sets a limit of players to this room. This property is shown in lobby, too. If the room is full (players count == maxplayers), joining this room will fail.
Name The name of a room. Unique identifier (per Loadbalancing group) for a room/match.
PlayerCount Only used internally in lobby, to display number of players in room (while you're not in).
