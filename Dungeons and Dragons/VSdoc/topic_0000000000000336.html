Properties
 
Name Description
AutoCleanUp Gets if this room uses autoCleanUp to remove all (buffered) RPCs and instantiated GameObjects when a player leaves.
CustomProperties Read-only "cache" of custom properties of a room. Set via Room.SetCustomProperties (not available for RoomInfo class!). (inherited from RoomInfo).
EmptyRoomTtl Room Time To Live. How long a room stays available (and in server-memory), after the last player becomes inactive. After this time, the room gets persisted or destroyed.
ExpectedUsers List of users who are expected to join this room. In matchmaking, Photon blocks a slot for each of these UserIDs out of the MaxPlayers.
IsLocalClientInside State if the local client is already in the game or still going to join it on gameserver (in lobby always false). (inherited from RoomInfo).
IsOpen Defines if the room can be joined. This does not affect listing in a lobby but joining the room will fail if not open. If not open, the room is excluded from random matchmaking. Due to racing conditions, found matches might become closed before they are joined. Simply re-connect to master and find another. Use property "visible" to not list the room.
IsVisible Defines if the room is listed in its lobby. Rooms can be created invisible, or changed to invisible. To change if a room can be joined, use property: open.
MasterClientId The ID (actorNumber) of the current Master Client of this room.
MaxPlayers Sets a limit of players to this room. This property is shown in lobby, too. If the room is full (players count == maxplayers), joining this room will fail.
Name The name of a room. Unique identifier (per Loadbalancing group) for a room/match.
PlayerCount Count of players in this room.
PlayerTtl Player Time To Live. How long any player can be inactive (due to disconnect or leave) before the user gets removed from the playerlist (freeing a slot).
PropertiesListedInLobby A list of custom properties that should be forwarded to the lobby and listed there.
