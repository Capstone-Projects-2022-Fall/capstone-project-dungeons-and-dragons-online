PhotonNetwork Properties

The following tables list the members exposed by the PhotonNetwork type.

Properties
 
Name Description
AuthValues A user's authentication values used during connect.
CloudRegion Currently used Cloud Region (if any). As long as the client is not on a Master Server or Game Server, the region is not yet defined.
CrcCheckEnabled Crc checks can be useful to detect and avoid issues with broken datagrams. Can be enabled while not connected.
CurrentCluster The cluster name provided by the Name Server.
EnableLobbyStatistics Set in PhotonServerSettings asset. Enable to get a list of active lobbies from the Master Server.
Friends Read-only list of friends, their online status and the room they are in. Null until initialized by a FindFriends call.
FriendsListAge Age of friend list info (in milliseconds). It's 0 until a friend list is fetched.
LobbyStatistics If turned on, the Master Server will provide information about active lobbies for this application.
MaxResendsBeforeDisconnect Defines the number of times a reliable message can be resent before not getting an ACK for it will trigger a disconnect. Default: 5.
NetworkStatisticsEnabled Enables or disables the collection of statistics about this client's traffic.
PacketLossByCrcCheck If CrcCheckEnabled, this counts the incoming packages that don't have a valid CRC checksum and got rejected.
PrefabPool An Object Pool can be used to keep and reuse instantiated object instances. It replaced Unity's default Instantiate and Destroy methods.
QuickResends In case of network loss, reliable messages can be repeated quickly up to 3 times.
ResentReliableCommands Count of commands that got repeated (due to local repeat-timing before an ACK was received).
Server The server (type) this client is currently connected or connecting to.
ServerAddress Currently used server address (no matter if master or game server).
ServerTimestamp The current server's millisecond timestamp.
UseAlternativeUdpPorts Switch to alternative ports for a UDP connection to the Public Cloud.
autoCleanUpPlayerObjects This setting defines per room, if network-instantiated GameObjects (with PhotonView) get cleaned up when the creator of it leaves.
autoJoinLobby Set in PhotonServerSettings asset. Defines if the PhotonNetwork should join the "lobby" when connected to the Master server.
automaticallySyncScene Defines if all clients in a room should load the same level as the Master Client (if that used PhotonNetwork.LoadLevel).
connected False until you connected to Photon initially. True in offline mode, while connected to any server and even while switching servers.
connectedAndReady A refined version of connected which is true only if your connection to the server is ready to accept operations like join, leave, etc.
connecting True when you called ConnectUsingSettings (or similar) until the low level connection to Photon gets established.
connectionState Simplified connection state
connectionStateDetailed Detailed connection state (ignorant of PUN, so it can be "disconnected" while switching servers).
countOfPlayers The count of players currently using this application (available on MasterServer in 5sec intervals).
countOfPlayersInRooms Count of users currently playing your app in some room (sent every 5sec by Master Server). Use PhotonNetwork.playerList.Length or PhotonNetwork.room.PlayerCount to get the count of players in the room you're in!
countOfPlayersOnMaster The count of players currently looking for a room (available on MasterServer in 5sec intervals).
countOfRooms The count of rooms currently in use (available on MasterServer in 5sec intervals).
gameVersion Version string for your this build. Can be used to separate incompatible clients. Sent during connect.
inRoom Is true while being in a room (connectionStateDetailed == ClientState.Joined).
insideLobby True while this client is in a lobby.
isMasterClient Are we the master client?
isMessageQueueRunning Can be used to pause dispatching of incoming evtents (RPCs, Instantiates and anything else incoming).
isNonMasterClientInRoom True if we are in a room (client) and NOT the room's masterclient
lobby The lobby that will be used when PUN joins a lobby or creates a game.
masterClient The Master Client of the current room or null (outside of rooms).
offlineMode Offline mode can be set to re-use your multiplayer code in singleplayer game modes. When this is on PhotonNetwork will not create any connections and there is near to no overhead. Mostly usefull for reusing RPC's and PhotonNetwork.Instantiate
otherPlayers The list of players in the current room, excluding the local player.
player The local PhotonPlayer. Always available and represents this player. CustomProperties can be set before entering a room and will be synced as well.
playerList The list of players in the current room, including the local player.
playerName Set to synchronize the player's nickname with everyone in the room(s) you enter. This sets PhotonNetwork.player.NickName.
room Get the room we're currently in. Null if we aren't in any room.
sendRate Defines how many times per second PhotonNetwork should send a package. If you change this, do not forget to also change 'sendRateOnSerialize'.
sendRateOnSerialize Defines how many times per second OnPhotonSerialize should be called on PhotonViews.
time Photon network time, synched with the server.
