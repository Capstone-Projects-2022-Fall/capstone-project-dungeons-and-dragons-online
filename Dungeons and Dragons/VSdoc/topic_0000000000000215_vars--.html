PhotonNetwork Fields

The following tables list the members exposed by the PhotonNetwork type.

Fields
 
Name Description
BackgroundTimeout Defines how many seconds PUN keeps the connection, after Unity's OnApplicationPause(true) call. Default: 60 seconds.
InstantiateInRoomOnly If true, Instantiate methods will check if you are in a room and fail if you are not.
MAX_VIEW_IDS The maximum number of assigned PhotonViews per player (or scene). See the [General Documentation](@ref general) topic "Limitations" on how to raise this limitation.
PhotonServerSettings Serialized server settings, written by the Setup Wizard for use in ConnectUsingSettings.
PrefabCache Keeps references to GameObjects for frequent instantiation (out of memory instead of loading the Resources).
SendMonoMessageTargetType Defines which classes can contain PUN Callback implementations.
SendMonoMessageTargets If not null, this is the (exclusive) list of GameObjects that get called by PUN SendMonoMessage().
StartRpcsAsCoroutine Can be used to skip starting RPCs as Coroutine, which can be a performance issue.
UsePrefabCache While enabled (true), Instantiate uses PhotonNetwork.PrefabCache to keep game objects in memory (improving instantiation of the same prefab).
UseRpcMonoBehaviourCache
logLevel Network log level. Controls how verbose PUN is.
maxConnections Obsolete. Only used in Unity Networking. In PUN, set the number of players in PhotonNetwork.CreateRoom.
precisionForFloatSynchronization The minimum difference between floats before we send it via a PhotonView's OnSerialize/ObservingComponent.
precisionForQuaternionSynchronization The minimum angle that a rotation needs to change before we send it via a PhotonView's OnSerialize/ObservingComponent.
precisionForVectorSynchronization The minimum difference that a Vector2 or Vector3(e.g. a transforms rotation) needs to change before we send it via a PhotonView's OnSerialize/ObservingComponent.
versionPUN Version number of PUN. Also used in GameVersion to separate client version from each other.
